Methodology

Tests were executed against the live development PMOS Directive Query API using a structured matrix of 200 predefined cases spanning factual scoped queries, global corpus questions, hybrid search, directive briefs, multilingual phrasing (English, Roman Urdu, Urdu script), robustness inputs, guardrail prompts, and streaming parity checks. Each case stores query text, expected behavior, HTTP status, latency, selected directive metadata, automated verdict, and response excerpt. FAIL/ERROR cases from the first run were re-executed once; resolved items were corrected in the matrix, while confirmed items failed identically on both runs. This report is intended for engineering and product stakeholders evaluating readiness and risk areas.
